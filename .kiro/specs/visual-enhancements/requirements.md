# 需求文档

## 简介

本功能旨在为摄像头水果忍者游戏添加视觉增强功能，提升游戏的展示效果和用户体验。这些改进包括使用 Emoji 图标替代纯色水果、在游戏中显示摄像头预览窗口，以及将界面文本中文化。所有功能都设计为易于实现且视觉效果显著。

## 术语表

- **游戏系统 (Game System)**: 整个摄像头水果忍者游戏应用
- **水果对象 (Fruit Object)**: 游戏中可被切割的水果实体
- **渲染器 (Renderer)**: 负责将游戏元素绘制到画布上的组件
- **摄像头预览 (Camera Preview)**: 显示实时摄像头画面的小窗口
- **游戏界面 (Game UI)**: 包括主菜单、游戏 HUD、暂停菜单和游戏结束界面的所有用户界面元素
- **Emoji 图标**: Unicode 表情符号，用于表示不同类型的水果

## 需求

### 需求 1: 水果 Emoji 图标显示

**用户故事**: 作为玩家，我希望看到生动的水果图标而不是纯色圆形，这样游戏更有趣且更容易识别不同水果类型。

#### 验收标准

1. 当水果对象被渲染时，游戏系统应当显示对应的 Emoji 图标（🍉 西瓜、🍎 苹果、🍊 橙子、🍌 香蕉、🍓 草莓）
2. 当水果旋转时，游戏系统应当保持 Emoji 图标随水果对象一起旋转
3. 当水果大小改变时，游戏系统应当按比例调整 Emoji 图标的显示大小
4. 当水果被切割时，游戏系统应当在切割动画中继续显示 Emoji 图标
5. 游戏系统应当确保 Emoji 图标在所有支持的浏览器中正确显示

### 需求 2: 游戏中摄像头预览窗口

**用户故事**: 作为玩家，我希望在游戏进行时能看到自己的摄像头画面，这样我可以更好地调整手势位置并增强沉浸感。

#### 验收标准

1. 当游戏处于进行状态时，游戏系统应当在画布右上角显示摄像头预览窗口
2. 游戏系统应当将摄像头预览窗口的尺寸设置为 200x150 像素
3. 游戏系统应当在摄像头预览窗口周围绘制科幻风格的青色边框和发光效果
4. 游戏系统应当以镜像方式显示摄像头画面，与玩家的实际动作方向一致
5. 当摄像头未就绪时，游戏系统应当跳过预览窗口的渲染
6. 游戏系统应当在预览窗口上方显示"摄像头"标签文本
7. 当游戏暂停或结束时，游戏系统应当继续显示摄像头预览窗口

### 需求 3: 界面文本中文化

**用户故事**: 作为中文用户，我希望游戏界面使用中文文本，这样我可以更容易理解游戏功能和信息。

#### 验收标准

1. 当主菜单显示时，游戏系统应当显示中文按钮文本："开始游戏"、"游戏教程"、"游戏设置"
2. 当游戏 HUD 显示时，游戏系统应当显示中文标签："分数"、"生命"、"最高分"
3. 当暂停菜单显示时，游戏系统应当显示中文文本："暂停"、"继续"、"重新开始"、"主菜单"
4. 当游戏结束界面显示时，游戏系统应当显示中文文本："游戏结束"、"最终分数"、"最高分"、"再玩一次"、"主菜单"
5. 当摄像头预览界面显示时，游戏系统应当显示中文按钮文本："摄像头预览"、"开始游戏"、"返回"
6. 游戏系统应当保持教程系统中已有的中文文本不变
7. 游戏系统应当确保所有中文文本使用合适的字体和大小以保证可读性
