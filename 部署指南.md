# 网页端水果忍者 - 部署指南

本指南介绍如何将项目部署到公网，提供多种免费和付费的部署方案。

## 📋 部署前准备

### 1. 构建项目

```bash
npm install
npm run build
```

构建完成后，需要部署的文件包括：
- `index.html`
- `dist/` 目录（编译后的 JS 文件）
- `node_modules/@mediapipe/` 目录（MediaPipe 依赖）

### 2. 重要提示

⚠️ **HTTPS 要求**：由于项目使用摄像头功能，必须在 HTTPS 环境下运行（localhost 除外）。所有推荐的部署平台都默认提供 HTTPS。

---

## 🚀 部署方案

### 方案一：Vercel（推荐 ⭐）

**优点**：免费、自动 HTTPS、部署简单、支持自动部署

#### 步骤：

1. **安装 Vercel CLI**
```bash
npm install -g vercel
```

2. **创建 vercel.json 配置文件**（已为你准备好）

3. **登录并部署**
```bash
vercel login
vercel
```

4. **后续更新**
```bash
npm run build
vercel --prod
```

#### 访问地址
部署成功后会得到类似 `https://your-project.vercel.app` 的地址

---

### 方案二：Netlify

**优点**：免费、易用、支持拖拽部署

#### 方式 A：通过网页界面

1. 访问 [netlify.com](https://www.netlify.com/)
2. 注册/登录账号
3. 点击 "Add new site" → "Deploy manually"
4. 将整个项目文件夹拖拽到页面上
5. 等待部署完成

#### 方式 B：通过 CLI

```bash
npm install -g netlify-cli
netlify login
netlify deploy --prod
```

选择部署目录时，选择项目根目录。

---

### 方案三：GitHub Pages

**优点**：完全免费、与 GitHub 集成

#### 步骤：

1. **将代码推送到 GitHub**
```bash
git init
git add .
git commit -m "Initial commit"
git remote add origin https://github.com/你的用户名/你的仓库名.git
git push -u origin main
```

2. **配置 GitHub Pages**
   - 进入仓库的 Settings → Pages
   - Source 选择 "main" 分支
   - 保存

3. **访问地址**
   - `https://你的用户名.github.io/你的仓库名/`

#### 注意事项
如果使用 GitHub Pages，可能需要调整资源路径。

---

### 方案四：Cloudflare Pages

**优点**：免费、速度快、全球 CDN

#### 步骤：

1. 访问 [pages.cloudflare.com](https://pages.cloudflare.com/)
2. 连接 GitHub 仓库
3. 配置构建设置：
   - Build command: `npm run build`
   - Build output directory: `/`
4. 部署

---

### 方案五：自己的服务器（VPS）

**适用场景**：已有服务器或需要完全控制

#### 使用 Nginx

1. **上传文件到服务器**
```bash
scp -r ./* user@your-server:/var/www/fruit-ninja/
```

2. **配置 Nginx**
```nginx
server {
    listen 80;
    server_name your-domain.com;
    
    # 重定向到 HTTPS
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name your-domain.com;
    
    ssl_certificate /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;
    
    root /var/www/fruit-ninja;
    index index.html;
    
    location / {
        try_files $uri $uri/ /index.html;
    }
    
    # 缓存静态资源
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
}
```

3. **获取 SSL 证书**（使用 Let's Encrypt）
```bash
sudo apt install certbot python3-certbot-nginx
sudo certbot --nginx -d your-domain.com
```

## 🔧 配置文件

### vercel.json
```json
{
  "version": 2,
  "builds": [
    {
      "src": "package.json",
      "use": "@vercel/static-build",
      "config": {
        "distDir": "."
      }
    }
  ],
  "routes": [
    {
      "src": "/(.*)",
      "dest": "/$1"
    }
  ]
}
```

### netlify.toml
```toml
[build]
  publish = "."
  command = "npm run build"

[[headers]]
  for = "/*"
  [headers.values]
    X-Frame-Options = "DENY"
    X-Content-Type-Options = "nosniff"
```

## ⚠️ 常见问题

### 1. 摄像头无法访问
- 确保使用 HTTPS
- 检查浏览器权限设置
- 确认域名在浏览器的信任列表中

### 2. MediaPipe 加载失败
- 确保 `node_modules/@mediapipe/` 目录已部署
- 检查网络连接
- 查看浏览器控制台错误信息

### 3. 资源 404 错误
- 检查 `index.html` 中的资源路径
- 确认所有依赖文件都已上传

### 4. 性能问题
- 启用 CDN 加速
- 压缩静态资源
- 使用浏览器缓存
